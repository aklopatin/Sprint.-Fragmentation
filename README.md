# Спринт: семантическая сегментация (Fragmentation)

Отчёт по проекту в виде форка mmsegmentation. Весь свой код и промежуточные результаты размещены в сабмодуле **practicum_work**.

---

## Условия проекта

- **Репозиторий:** работа выполняется в **форке mmsegmentation** (обучение, валидация и инференс — через конфиги и скрипты этого форка).
- **Целевая метрика:** **mDice** (mean Dice score).
- **Критерий выполнения:** проект считается выполненным при **mDice > 0.75**.

---

## Отчёт о проделанной работе

### Датасет

- **Путь:** `train_dataset_for_students/`
- **Структура:** `img/{train,val,test}/` — изображения `.jpg`, `labels/{train,val,test}/` — маски `.png` (те же имена).
- **Классы:** 3 (метки 0, 1, 2).
- EDA датасета выполнен в ноутбуке `eda_dataset.ipynb` с использованием кода из `practicum_work/src/data`.

### Модель и обучение

- **Модель:** DeepLabV3 (ResNet50), конфиг: `configs/deeplabv3_fragmentation.py`.
- Валидация и отбор модели — по метрике **mDice** (настроено в `configs/fragmentation_dataset.py`).
- Подробные инструкции по окружению, обучению и инференсу: **README_MMSEG.md**.

## Этап 5. Документация кода

Схема структуры кода в сабмодуле и краткое описание скриптов:

```
practicum_work
├── src
│   ├── data
│   │   ├── dataset_utils.py
│   │   │   — загрузка пар img/mask, классы, пиксели по классам, размеры изображений, площади объектов
│   │   ├── class_balance.py
│   │   │   — баланс классов по пикселям, круговые диаграммы для train/val
│   │   ├── image_sizes.py
│   │   │   — статистика размеров изображений, гистограммы и scatter W/H
│   │   ├── object_sizes.py
│   │   │   — площади объектов по классам, гистограммы площадей (log10)
│   │   ├── domain_issues.py
│   │   │   — проверка несовпадения размеров img/mask, отсутствия классов, мелких объектов, только фон; итоговая сводка EDA
│   │   ├── check_num_classes.py
│   │   │   — проверка числа уникальных меток в масках (рекомендуемое num_classes)
│   │   └── __init__.py
│   ├── analysis
│   │   ├── parse_training_log.py
│   │   │   — парсинг логов MMEngine (iter, loss, mDice, mIoU); поиск чекпоинта best_mDice_iter_*.pth
│   │   ├── plot_training_curves.py
│   │   │   — построение графиков loss и mDice/mIoU по итерациям, сохранение в файл
│   │   ├── dice_per_sample_best_worst.py
│   │   │   — Dice на каждом семпле валидации (инференс по чекпоинту), топ-K и худшие K по Dice, запись экспериментов в experiments_log.json
│   │   ├── export_report_pdf.py
│   │   │   — экспорт ноутбука (ipynb) в PDF через nbconvert (webpdf или pdf)
│   │   └── __init__.py
├── supplementary
│   └── viz
│       — визуализации для отчёта: графики обучения, отрисовки семплов (топ-5/худшие 5), скриншоты
└── README.md
```

Все вспомогательные скрипты расположены в **practicum_work/src**: в **data** — код для первого этапа (EDA, статистики, проверка формата); в **analysis** — скрипты для анализа качества (метрики, парсинг логов, лучшие/худшие предсказания, экспорт отчёта). Визуализации для отчёта размещаются в **practicum_work/supplementary/viz** (допускается произвольная вложенность).

В корне репозитория также используются конфиги `configs/`, скрипты `scripts/` (хуки mmseg, логирование для запуска из ноутбука), `run_train.py`, `run_inference.py` и ноутбук `eda_dataset.ipynb`, которые вызывают или импортируют функциональность из practicum_work и из scripts.
